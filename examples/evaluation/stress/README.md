# 风险审查重排序压力集

本目录包含一份 50 条采购合同和一份 100 条制度，用于比较向量排序与重排序的差异。数据完全虚构，不代表真实企业制度或法律意见。

## 数据目标

- 正式付款条款位于 `C018`，周围包含保证金、报销、历史合同、草案和阶段款等高相似内容。
- 正式质保条款位于 `C030`，周围包含维护期、寿命、软件升级期、消耗品例外和宣传资料等内容。
- 正式违约责任位于 `C040`，周围包含不可抗力、保险、保密、延期和责任上限等内容。
- 正式争议解决位于 `C050`，前面放置签订地、交付地、谈判地和通知地址等地点干扰项。
- 对应制度主规则分别位于 `P025`、`P050`、`P075`、`P100`。

## 使用方式

建议在干净的演示数据库中只导入本目录制度和合同，等待两份文档向量化完成后发起审查。若同时导入小型评测集，其他当前制度也会进入制度候选池，适合做更高难度测试，但不便于复现实验基线。

重排序实验建议统一使用：

```text
合同向量召回 Top 20 → 重排序 → 最终 Top 3
制度向量召回 Top 30 → 重排序 → 最终 Top 5
```

基线组使用同一批候选但不重排序，直接截取向量排名前 3/5。对比 `MRR`、`Precision@K`、`NDCG@K`、最终引用准确率和风险状态准确率。
